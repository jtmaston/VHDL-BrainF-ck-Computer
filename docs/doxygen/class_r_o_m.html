ROM Entity Reference

Detailed Description

ROM Entity. This essentially implements a ROM chip, that contains a number of initialized, read-only cells. This model is based on transasctions: falling edges of transact cause changes to the data lines. Upon hitting the rising edge of transact, data lines are returned to 'Z' (i.e. cleared for another component to use.) Usage example: ROM_tb.vhd

Example wavefile:

Precondition
Address must be loaded on addrLines before the falling edge of transact
Postcondition
Data lines can be considered valid at any time after @falling_edge of transact
